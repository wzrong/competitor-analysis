# 竞品分析知识库站点增加问题反馈系统

## Context

学科网情报系统站点（MkDocs Material + GitHub Pages）已对外开放，希望增加用户反馈入口，收集内容纠错、补充竞品和功能建议三类反馈。选择 GitHub Issues 方案，零成本、无需后端。

## Implementation

### 步骤 1：新增 3 个 GitHub Issue 模板

在 `website/.github/ISSUE_TEMPLATE/` 下新增：

- `content-fix.yml` — 内容纠错模板（标签 `content-fix`）
	- 问题位置、错误描述、正确信息、信息来源
- `add-competitor.yml` — 补充竞品模板（标签 `add-competitor`）
	- 竞品名称、官网 URL、产品类型、一句话描述、关注理由
- `feature-request.yml` — 功能建议模板（标签 `feature-request`）
	- 建议类型、具体描述、预期价值

### 步骤 2：新建反馈页面

在 `website/docs/feedback/index.md` 创建反馈页面：

- 页头欢迎语
- 三个卡片（带 emoji 图标）：内容纠错、补充竞品、功能建议
- 每个卡片带按钮跳转到对应 Issue 模板
- 页尾加「浏览所有反馈」链接到 Issues 列表

### 步骤 3：更新导航配置

修改 `website/mkdocs.yml`，在 nav 末尾增加：
```
- 反馈: feedback/index.md
```

## Verification

1. 本地启动站点预览，确认导航栏出现「反馈」标签
2. 访问反馈页面，确认三个卡片显示正常、图标正确
3. 点击每个按钮，确认跳转至正确的 GitHub Issue 模板（URL 含 `template=xxx.yml`）
4. 检查 Issue 模板表单字段是否正确渲染
5. 构建并部署站点
