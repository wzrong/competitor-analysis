# 竞品分析框架

每个竞品分析可从以下四个维度切入。做**单竞品深度分析**时按此框架逐一覆盖；做**多竞品对比**时按此框架横向比较。

---

## 维度一：产品功能对比

**核心问题**：这个竞品做了什么？和我们比少了什么、多了什么？

**分析要点：**

- 核心功能列表及实现方式
- 与我们产品的功能重叠度（重叠/互补/无交集）
- 独有功能（我们没做但它做了的）——是否值得跟进？
- 缺失功能（我们做了但它没做的）——是取舍还是还没做？
- 功能更新频率——最近上线了哪些新功能？

**输出格式：**

- 功能对比矩阵（竞品1/竞品2/.../我们）
- 每个差异点的值得关注度标注（高/中/低）

---

## 维度二：用户体验/交互流程

**核心问题**：用户用得顺不顺？我们的流程哪里不如它？

**分析要点：**

- 核心任务路径（从进入到完成关键操作的步骤数）
- 界面布局与信息架构
- 新手引导与帮助体系
- 视觉风格与品牌调性
- 响应速度与性能体感
- 移动端/多端体验一致性

**输出格式：**

- 关键流程截图对比（标注差异点）
- 流程步骤数对比
- 体验亮点和痛点总结

---

## 维度三：目标用户与定位

**核心问题**：它面向谁？定位和我们差多少？

**分析要点：**

- 目标用户画像（学段/学科/教师/学生/家长）
- 核心价值主张（一句话解决了什么问题）
- 定价模式与付费意愿
- 市场切入点（从哪个场景切入的）
- 品牌调性与话术风格
- 用户口碑与评价（搜索到的用户反馈）

**输出格式：**

- 定位一句话总结
- 用户群体对比表
- 定价对比（如有）

---

## 维度四：技术能力与数据

**核心问题**：底层实力怎么样？

**分析要点：**

- AI 模型/技术栈（是否有公开信息）
- 数据来源（教材/题库/自研）
- 准确率/可靠性（如有公开数据或用户反馈）
- 技术亮点或技术壁垒
- API 开放能力（是否提供对外接口）
- 技术团队的公开信息

**输出格式：**

- 技术栈对比表
- 已知技术能力和局限性总结
- 技术信息来源标注（官网/媒体报道/推测）

---

## 使用说明

- **单竞品深度分析**：四个维度全部覆盖，每个维度约一段分析 + 必要截图。
- **多竞品快速对比**：按用户指定的维度，用表格横向对比，每个维度一段总结。
- **特定维度定向分析**：用户指定时，只做该维度，但可以更深入（搜更多资料、截更多页面）。
