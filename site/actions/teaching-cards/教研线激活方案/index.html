    教研线 · 激活方案 v1.0¶

    本文档解决教研线在战略情报系统中"被定义了但从未被激活"的问题。
    覆盖：定位校准、触发标准、产出模板、历史欠账、与现有系统衔接。

    一、问题诊断¶

    教研线在系统中处于"三层问题叠加"状态：

    系统设计层：INDEX.md 定位描述写的是"3层价值输出（决策层/产品线/市场线）"，漏掉了教研线。CLAUDE.md 的 L9 明明定义了 9A/9B/9C/9D 四种受众，教研线从设计上就存在，但从索引开始就被遗漏了。

    竞品档案层：决策层简报/Battlecard/销售话术卡各已产出 3 家（21世纪教育网、九章爱学、飞象老师），教研参考卡一家都没出。应对建议/教研参考卡/ 目录是空的。

    政策分析层：Phase 2 产出中，"人工智能+教育"行动计划明确写了"循证教研"，但深度解读只覆盖了产品/市场/决策三条线，没有从教研角度展开。

    根因：教研线缺少自然推动力。另外三条线有明确的节奏——决策层简报（季度/重大变化）、Battlecard（月度）、销售话术卡（随时可用），它们和竞品监测这个常规动作挂钩。教研参考卡没有挂钩常规动作，全靠手动触发，所以即使竞品在做教研动作，也从未产出。

    二、定位与受众¶

    教研参考卡是 L9 四种产出之一，回答的问题：竞品的教研逻辑是什么、哪些值得学科网借鉴、哪些不需要跟。

    受众：学科网的教研团队——内容研发、课程设计、教材教辅团队。

    和 Battlecard 的分工：Battlecard 面向产品线和市场线，讲功能对比和竞争优势。教研参考卡面向内容线，讲教学逻辑、方法论和内容策略。同一个竞品动作——比如飞象老师 2.0 上线"AI教案·大单元"——Battlecard 的视角是"他们加了教案功能，我们 AI 研修跟不跟"，教研参考卡的视角是"他们的大单元设计方法论体现了什么课标取向、学科网的教研体系是否覆盖了这个方向"。

    三、触发标准¶

    每次遇到以下情况，必须出教研参考卡。内容多时详细展开，内容少时简化（至少包含教研逻辑拆解和建议行动两个核心板块），但不能没有。

    必须出¶

    • 竞品发布新课程体系或内容框架，与学科网核心学科重叠
    • 例：飞象老师上线"AI教案·大单元"是教学设计方法论的结构性变化，不是一个小功能点
    • 竞品调整核心教学法方向，涉及新课标对齐、素养导向转型、跨学科融合等结构性变化
    • 竞品上线对教研方向有直接冲击的 AI 功能
    • 例：AI 命题改变了传统命题工作流，涉及学科网题库业务的教研逻辑
    • 竞品与出版社/课标组/教研机构达成深度合作
    • 例：金太阳依赖教辅出版渠道进入学校，"教研背书+渠道绑定"模式需要专门分析
    • 竞品在特定学科的 UGC 内容量、质量、教学法分布出现明显结构性变化
    • 例：飞象老师社区数据中数学精选率 30.9%、英语热门率 0.6% 但精选率 19.6%，反映官方内容质量引导策略
    • 政策推动教研方向变化
    • 例：行动计划中"循证教研""教师智能素养标准""AI 纳入教师资格考试"三条与教研线直接相关
    • 竞品在教学法层面有创新实验
    • 例：游戏化教学、项目式学习、数据回收型互动课件等新模式

    不出¶

    • 常规 UI/UX 优化，不涉及内容策略和教学法
    • 营销活动、积分活动等纯运营类变化
    • Tier 2 竞品的常规迭代（除非该迭代触发 Tier 1 升级信号）

    四、产出模板¶

    # {竞品名} 教研参考卡

> 更新日期：YYYY-MM-DD | 触发事件：{一句话描述}

---

## 一、教研动向

具体描述触发事件。不是泛泛说"他们更新了课程体系"，而是具体到：做了什么、涉及哪些学科/学段、变化前后的差异。

## 二、教研逻辑拆解

回答"他们为什么这么做"。

### 2.1 教学理念取向
他们背后相信什么？是知识本位还是素养导向？是单课时思维还是大单元设计？是教师主导还是学生中心？给出判断依据——不是猜测，是从产品设计/内容组织/营销话术中提炼。

### 2.2 课标/政策依据
他们引用了哪版课标？对标了哪个政策方向？是 2022 版义教课标的"教学评一体化"还是高中的"核心素养"？找对依据才能判断这个方向是短期策略还是长期布局。

### 2.3 方法论特征
他们的内容生产逻辑是什么？例如飞象老师是 UGC 社区模式+AI 辅助生成，金太阳是教辅出版基因+校本化定制。不同的方法论决定不同的内容质量控制方式和更新速度。

## 三、与学科网教研对比

| 维度 | 学科网当前状态 | 竞品当前状态 | 差距判断 |
|---|---|---|---|
| 内容覆盖 | | | |
| 教学法取向 | | | |
| 内容更新机制 | | | |
| 质量把控 | | | |

## 四、借鉴价值判断

### 值得借鉴的
- [具体内容/方向]：[为什么值得学，与学科网教研体系的哪个环节可以对接]
- [具体内容/方向]：[为什么]

### 不需要跟的
- [具体内容/方向]：[为什么和学科网定位不符、或资源投入产出比低]
- [具体内容/方向]：[为什么]

| P1-本季度 | [具体动作] | | |
| P2-观察 | [什么情况下再启动] | | |

---

> 触发事件来源：[T级别, 来源, 日期]
> 下次评估：YYYY-MM-DD（或"[竞品名]下次重大教研动向时"）

    简化版（信息不足时使用）¶

    当触发事件信息有限时，保留以下核心板块即可：
    - 教研动向（有什么就写什么）
    - 教研逻辑拆解（至少 2.1 教学理念取向 + 2.2 课标/政策依据）
    - 建议教研行动（至少 P0 一条）
    - 不需要跟的（至少列一条明确不跟的）

    五、与现有系统的衔接¶

    与竞品监测（每周）的联动¶

    每周监测中发现竞品教研类变化时，不在简报中展开教研分析（简报是给产品/市场看的），而是标记"建议出教研参考卡"，另行产出。

    CLAUDE.md 修改点：在"每周监测"流程第 2 步 c（评估影响级别）之后，新增一条：

    f. 如果变化涉及教学法/课程体系/教研合作/内容策略，标记"建议出教研参考卡"

    与发版追踪（每两周）的联动¶

    发版追踪发现的功能变化，如果涉及内容策略和教学法层面，触发教研参考卡。

    CLAUDE.md 修改点：在"发版追踪"输出格式的"变化级别"之后，新增一条：

    如变化涉及内容策略或教学法层面，标记"建议出教研参考卡"

    与政策分析的联动¶

    政策解读中发现教研方向类条款，触发教研参考卡。

    政策分析 CLAUDE.md 修改点：在深度解读触发条件中新增一条：

    政策直接涉及教研方向/教师素养标准/教学内容标准 → 同步触发教研参考卡

    与档案维护的联动¶

    教研参考卡的结论反向写入竞品 profile.md。如果某竞品连续两次教研参考卡都判定"不需要跟"，在分层总表中标注该竞品在教研维度不构成威胁。

    六、历史欠账¶

    以下四条应补出教研参考卡，按执行顺序排列：

    1. 猿辅导-飞象老师（🚨，数据最充分，作为第一条样本）

    关键教研问题：教辅出版基因如何转化为数字化教研能力。
