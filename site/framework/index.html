    学科网战略情报系统 · 落地方案 v1.0¶

    本文档是竞品分析工作台升级为战略情报系统的完整执行蓝图。
    涵盖：系统架构、9层分析框架、竞品分层管理、四大模块实施规划、CLAUDE.md 改造方案、档案模板、L9输出规范、实施路线图。

    一、系统定位¶

    核心定位：学科网的战略导航仪——回答三个问题：风向往哪吹、钱往哪里流、我们往哪打。

    系统架构：1个定位 + 4大模块 + 5层价值输出

    学科网战略情报系统
├── 模块一：行业分析（趋势判断）
├── 模块二：竞争分析（竞品洞察）  ← 现有竞品分析工作台升级而来
├── 模块三：政策分析（合规与机会）
└── 模块四：市场监测（资金流向）

输出对象（5层，按受众分发，不能混用）：
├── 决策层   → 9A 战略简报（季度/重大变化时）
├── 产品线   → 9B Battlecard（月度更新）
├── 市场线   → 9C 销售话术卡（随时可用）
├── 教研线   → 9D 教研参考卡（竞品教研重大动向时触发）
└── 运营线   → 9E 运营参考卡（竞品运营动作重大变化时触发）

    与现有工作台的关系：在现有竞品分析工作台基础上扩展，不另起炉灶。现有22个竞品档案、监测日志、分析框架全部保留并升级。

    二、9层分析框架（完整规格）¶

    这是整个系统的分析内核。每个竞品的深度分析都按9层执行，浅度跟踪的可只覆盖L1-L5。

    L1 · 基本面¶

    定位：这家公司是谁。

    分析要点：

    • 公司全称、成立时间、注册地、实际运营地
    • 股权结构（母公司/集团背景，如金太阳→三端科技）
    • 融资历史（轮次、金额、投资方、估值）
    • 核心团队（创始人背景、关键高管、教育/技术复合背景）
    • 员工规模与组织架构
    • 核心资源（数据资产、专利、品牌、独家内容授权）

    • 商业模式（盈利方式：SaaS/项目制/买断/平台抽成）

    • 商业模式画像（⭐核心，是理解对方所有动作的底层框架）

    • 核心收入来源（卖什么：资源/SaaS/项目制/硬件/数据服务）
    • 客户获取逻辑（靠什么打开门：品牌/渠道/价格/教辅/政府关系）
    • 增长飞轮（什么在驱动规模扩张：内容积累/渠道下沉/平台网络效应）
    • 防御壁垒（凭什么留住客户：转换成本/数据沉淀/生态锁定）
    • 当前战略重心（现在在押注什么：AI/区域扩张/硬件/C端）

    商业模式画像的作用：每次分析竞品的新动作（发布功能/降价/合作/融资），都要先问"这个动作匹配他们的商业模式吗"。匹配的说明是战略级推进，值得重视；不匹配的可能是试探或跟风，不必过度解读。

    数据来源：企查查/天眼查、官网、公开融资报道、招聘信息（判断团队方向）、财报

    信源质量标注：T1=公开可查证，T2=行业报告，T3=一手调研

    L2 · 产品纵深¶

    定位：他们卖什么、怎么做的。

    分析要点：

    • 产品线全景（主产品/子产品/开放平台）
    • 核心功能清单（逐一列举，标注与学科网的重叠/差异）
    • 技术架构与AI能力（大模型能力、自研/外采、本地化部署能力）
    • 多端覆盖情况（PC/移动/小程序/大屏客户端）
    • 产品迭代节奏（版本更新频率、重大功能发布记录）
    • 用户体验评价（来自官方App评分、教师论坛、一线反馈）
    • 内容/资源体量（若有，与学科网做数量级对比）

    数据来源：产品官网、应用商店、用户评价平台、实测截图

    L3 · 价格力¶

    定位：他们怎么收钱，贵不贵，值不值。

    分析要点：

    • 定价模式（按校/按区/按人数/按功能模块/买断/订阅）
    • 价格区间（单校价格、区域价格、集采价格，尽可能量化）
    • 付费路径（免费试用→付费的转化机制，是否有freemium层）
    • 捆绑销售策略（是否与硬件、教辅、服务打包）
    • 与学科网的价格位置关系（贵/便宜/相当，在哪个场景下）
    • 招投标中标单价（来自市场监测模块的数据）

    数据来源：定价页面、招投标公开数据、销售一线反馈（T3）

    L4 · 渠道力¶

    定位：他们怎么触达客户、签下合同。

    分析要点：

    • 直销vs渠道（直销团队规模、代理商/集成商体系覆盖范围）
    • 与教育局/教研室的关系深度（是否有政府背书、战略合作协议）
    • 招投标能力（入围资质、历史中标记录、中标区域分布）
    • 硬件捆绑渠道（是否与大屏厂商、平板厂商绑定销售）
    • 内容/品牌渠道（教辅品牌背书，如金太阳靠教辅书打开学校门）
    • 线上获客（官网SEO、公众号粉丝量、广告投放）
    • 合作生态（与哪些平台、出版社、机构有深度合作）

    数据来源：招投标平台、官网合作伙伴页面、行业媒体报道、销售一线情报（T3）

    L5 · 市场表现¶

    定位：他们实际打下了多大市场。

    分析要点：

    • 合作学校数量（绝对值 + 覆盖学段）
    • 注册/活跃用户数（教师端/学生端分开统计）
    • 区域分布（哪些省份是核心阵地，对比学科网的弱势区域）
    • 招投标历史数据（项目数量、累计金额、核心区域）
    • 市场份额（若有行业数据，注明口径）
    • 增长趋势（同比增速，是否在快速扩张）
    • 品牌认知度（行业提及频率、教师圈口碑）

    数据来源：公司官网/PR稿、行业白皮书、招投标平台监测

    L6 · SWOT¶

    定位：综合判断，必须联动学科网视角——不是单纯分析对手，而是找到对我们的启示。

    分析要点：

    • S 优势：对方哪里真的做得好，客观承认（不能只写缺点）
    • W 劣势：对方真实的短板，有数据支撑
    • O 机会：外部趋势对对方有利的地方（如政策利好、资本进入）
    • T 威胁：对方面临的市场/技术/政策风险

    每条SWOT必须注明「对学科网的含义」，不能只写对手分析。

    L7 · 战略卡位¶

    定位：这家公司在赛道里站在哪个位置，和我们的交叉点在哪。

    分析要点：

    • 赛道定位（主赛道 + 延伸赛道）
    • 竞争关系分类（🔴核心直接竞品/🟡直接竞品/🟢间接竞品）
    • 与学科网的重叠场景（具体列出：备课/组卷/阅卷/学情/研修等）
    • 与学科网的差异化（他们有、我们没有的；我们有、他们没有的）
    • 目标客户重叠度（是否在争同一批学校、同一批采购预算）
    • 竞争强度评级（🔥激烈/⚡上升/💤稳定）

    L8 · 战略研判¶

    定位：他们下一步会干什么，为什么。这是最难也最有价值的层。

    分两个子维度：

    8A · 动向研判（他们会做什么）：

    • 产品迭代方向（根据当前重点投入判断，招聘岗位是重要信号）
    • 市场扩张方向（当前空白区域、计划进入的新赛道）
    • 融资/并购动向（资金状况、是否有整合预期）
    • 合作动态（正在谈/已签的战略合作意味着什么）

    8B · 管理层假设（他们相信什么）：

    • 他们认为自己的核心优势是什么（从对外发言、产品命名、营销话术中提炼）
    • 他们对学科网的认知偏差在哪里（低估/高估了哪些方面）
    • 他们的战略盲点（资源分配倾斜导致的顾此失彼）
    • 他们在什么情况下会发动反攻（触发条件）

    数据来源：创始人/CEO公开发言、招聘岗位变化、产品发布节奏、行业峰会PPT

    L9 · 应对建议¶

    定位：情报的终点是行动，不是报告。五种受众，五种格式，不能混用。

    9A · 决策层战略简报（1页，每季度或重大变化时触发）：

    标题：【竞品名】战略简报 YYYY-QN
核心结论：1句话（威胁等级 + 建议动作）
威胁等级：🚨高 / ⚠️中 / ℹ️低（附判断依据）
关键变化：本季度3条最重要的情报
战略建议：针对决策层的2-3个行动选项（要/不要/观望，各附理由）
风险提示：如果不行动，最坏情况是什么

    9B · 产品线 Battlecard（1-2页，月度更新）：

    竞品：[名称] vs 学科网
核心差异（3条）：我们更强的 / 他们更强的
常见对比场景：当客户说「XX比你们好」时怎么回
我们的差异化话术：2-3句核心话术
功能对比表：关键功能 ✅/❌ 对比
数据弹药：可以引用的数据（资源量、覆盖学校、更新速度）

    9C · 市场线销售话术卡（半页，随时可用）：

    竞品：[名称]
他们的销售话术（已知）：他们通常怎么攻击我们
我们的反制话术：逐条回应
问题设置（trap questions）：引导客户意识到对方弱点的问法
绝对不要说的话：会踩雷的话

    9D · 教研线参考卡（按需生成，竞品有重大教研动向时触发）：

    竞品：[名称]
教研动向：[本次变化描述]

不需要跟的：[为什么，和学科网定位不符的地方]

    说明：教研线针对教研团队（如张慧红）。触发条件：竞品发布新课程体系、调整内容框架、上线新教研工具、与课标政策联动的教学法变化。保存路径：应对建议/教研参考卡/

    9E · 运营线参考卡（按需生成，竞品有重大运营动作时触发）：

    竞品：[名称]

  - 可以学的：[具体机制/策略/活动设计]
  - 不适合学科网的：[原因，如用户群差异/商业模式不同]
建议运营行动（P0/P1/P2分级）：[具体可操作的建议]

    说明：运营线针对内容运营/用户运营团队。触发条件：竞品推出新会员/订阅体系、重大促销活动、用户增长机制变化、内容运营策略调整（如免费资源扩充、签到/积分/激励体系上线）。保存路径：应对建议/运营参考卡/（目录待创建）

    三、竞品分层管理规则¶

    三级分层标准¶

    按「学科网业务威胁度 × 公司体量」双轴划分：

    Tier 1 · 核心战略竞品（15家上限）

    • 标准：与学科网核心产品直接竞争 + 体量不可忽视
    • 分析深度：完整9层框架
    • 监测频率：每周监测，有变化立即更新
    • 档案维护：profile.md全字段 + analyses/ 分析记录 + screenshots/
    • 当前建议纳入：21世纪教育网、菁优网、橡皮网、教习网、正确云、希沃、好未来-九章爱学、猿辅导-飞象老师、智学网、作业帮教师版，以及从赛道总表中补入的精准教学/综合平台赛道头部

    Tier 2 · 重点跟踪竞品（40家上限）

    • 标准：赛道相邻 或 正在快速扩张进入学科网领地
    • 分析深度：L1-L5（基础面+市场层），有重大动态时升级至9层
    • 监测频率：月度监测
    • 档案维护：profile.md核心字段 + 有变化时的分析记录
    • 来源：现有🟡🟢竞品 + 赛道总表各赛道TOP3

    Tier 3 · 观察池（150+家）

    • 标准：赛道总表中其余企业，暂无直接威胁
    • 分析深度：仅维护基础标签（公司名/赛道/子分类/核心产品）
    • 监测频率：季度扫描（关键词：行业黑马、新融资、异常扩张）
    • 档案维护：不建profile档案，在观察池INDEX中维护一行记录

    升降机制¶

    Tier 3 → Tier 2：发现以下任一信号
  - 获得B轮及以上融资
  - 单季度中标金额超过100万
  - 开始在学科网核心区域批量扩张
  - 发布明显对标学科网的产品功能

Tier 2 → Tier 1：连续两次月度监测发现战略级变化
  - 产品功能完成对学科网全场景覆盖
  - 与头部硬件厂商/出版集团达成独家合作
  - 被科大讯飞/好未来/字节等大厂收购或深度绑定

Tier 1 → Tier 2：连续6个月无重大动态，威胁等级持续降低

    五维企业标签体系¶

    每个竞品建立5维标签，用于快速筛选和对比：

    [赛道] 精准教学/资源备课/智慧作业/综合平台/硬件/其他
[阶段] A轮前/A-B轮/B轮后/上市/无融资记录
[地域] 全国/华东/华南/华北/华中/西部

    五、数据质量与信源规范（于晓媚反馈）¶

    于晓媚的担忧直接指向了一个核心问题：填了不可信的数据，比不填更危险。本章节定义强制执行的质量规范。

    5.1 信源质量三级标准¶

    级别 定义 举例 可信度
    T1 公开可查证的一手来源 官网、招投标公告、企查查、App截图 高，直接使用
    T2 有来源的二手信息 行业白皮书、媒体报道、分析师报告 中，需注明来源
    T3 一手调研/内部信息 销售反馈、用户访谈、竞品员工信息 高但难验证，需注明来源和时间

    5.2 不确定信息处理规则（于晓媚④）¶

    AI遇到以下情况，必须主动向用户提问，不能默默填入或猜测：

    • 搜索结果互相矛盾，无法判断哪个准确
    • 数据明显过时（超过6个月），但无更新来源
    • 信息来源不明，无法归入T1/T2/T3
    • 关键字段（如价格、用户规模、融资金额）只有估算值

    提问格式：

    ⚠️ 数据待确认：[字段名]
当前找到的信息：[描述]
来源：[来源] | 可信度：[T级别]
问题：[具体需要用户确认的内容]
如果无法确认，此字段将标注「待核实」，不填入具体数值。

    5.3 不可用内容处理规则（于晓媚②）¶

    以下情况的内容不得填入档案，必须标注「待核实」：

    • 无法找到可靠来源（无法归入任何T级别）
    • 信息明显是竞品自我宣传，缺乏第三方验证（如竞品官网声称"全国第一"但无数据支撑）
    • 数据超过12个月且行业变化快

    档案中出现「待核实」字段时，在每次监测时优先尝试补全，补全后注明来源和日期。

    5.4 内容可信度标注规范¶

    所有档案字段后如有数据，必须标注来源和质量级别，格式：

    用户规模：200万+教师用户 [T2, 艾瑞咨询2025年白皮书, 2025-03]
合作学校：5000+ [T1, 官网首页, 2026-05-20]
融资情况：B轮5亿估值 [T2, 36kr报道, 2025-06-15] ⚠️待核实（无官方确认）

    Phase 0（当前会话）：确认方案¶

    • 用户确认本落地方案

    Phase 1（P0，1-2周）：竞争分析框架升级¶

    • 更新竞品分析工作台 CLAUDE.md（9层框架 + 分层规则 + L9规范）
    • 更新 profile.md 模板
    • 现有22个竞品重新分层（Tier ½）
    • Tier 1竞品档案补全L3/L4/L8/L9（逐个执行）
    • 建立观察池 INDEX（从赛道总表导入）

    Phase 2（P1，3-4周）：政策分析模块¶

    • 建立政策分析目录结构
    • 建立 policy_timeline.md 历史政策录入
    • 确定政策信源列表
    • 首次政策解读（"AI+教育"行动计划）

    Phase 3（P2，信源确认后）：行业分析模块¶

    • 与用户确认信源体系
    • 建立行业分析目录结构
    • 首次手动运行，验证格式
    • 转为定时任务

    Phase 4（P3，单独规划）：市场监测模块¶

    • 调研招投标数据采集方案
    • 建立市场监测目录结构
    • 技术方案确认后实施

    Phase 5（并行推进）：发版追踪机制¶

    • 确定Tier 1竞品的核心页面清单（每家3-5个页面）
    • 建立基准截图（首次全量截图，作为比对基线）
    • 建立发版追踪目录 竞品库/{竞品名}/发版追踪/
    • 接入每两周定时任务（Phase 2完成后配置）
    • 验证截图比对的输出质量，人工校准一轮

    Phase 6（中期规划）：Skill化路径¶

    张慧红⑥的建议：把优秀的分析流程固化成Skill，让AI工具可以标准化执行。

    路径规划：
    1. 先积累：Phase 1-3期间，积累高质量的人工分析样本（金太阳报告是第一个标杆）
    2. 再提炼：对已有分析样本做方法论提炼，定义每个步骤的输入/输出/判断规则
    3. 再固化：将提炼出的方法论写入 Skill 指令，配合自动化工具执行
    4. 再调教：每次AI产出后，人工审核+标注问题，持续优化Skill

    预计在Phase 1-2完成、积累5个以上高质量分析样本后，启动Skill化工作。
